# Frameworks

The named ideation engines prolific creators use. Each one is a way of
injecting a constraint so the brain can triangulate instead of
free-associating. Pick the one that fits the brief, run it end to end,
and produce labeled output. Don't blend three at once.

## Justin Welsh's content matrix (when you want volume in one sitting)

A 2D grid. Down one axis: 5 to 10 sub-topics inside the niche. Across
the other: 5 to 7 named styles. Every cell is an idea.

Welsh's canonical style set:
- **Actionable.** Ultra-specific how-to.
- **Motivational.** Industry or personal stories.
- **Analytical.** Break down a topic with reasoning.
- **Contrarian take.** State a defensible position against the consensus.
- **Observation.** Notice something most people miss.
- **Prediction.** Make a forward call.
- **Step-by-step guide.** Numbered sequence.
- **Inspirational story.** A lesson through a person.

Math: 5 sub-topics x 8 styles = 40 distinct cells. Each cell can produce
three to four pieces of content (a short, a carousel, a thread, a long
post). One filled matrix is a quarter of programming.

Use it when: the user wants 20+ ideas now, or is starting a content
calendar from zero. The `assets/content-matrix.md` template is the
filled-in version.

## Ship 30's atomic-essay angle set (when you want to turn one topic into many)

Originally written for short-form text by Dickie Bush and Nicolas Cole.
The mechanic transfers cleanly to short-form video. Pick one topic, run
it through five angles, get five non-repeating pieces.

The five angles:
- **Consumption-based.** Write about what you just read, watched, or used.
- **Response-based.** React to or extend someone else's piece.
- **Curation.** A "best of" roundup pulled from your domain.
- **Instructional.** A how-to on a sub-skill.
- **Narrative.** A personal lesson with a takeaway.

Use it when: the user has one strong topic and wants 5 angles, or the
user is trying to milk a single high-performer for derivative posts.
See `repurposing.md` for the longer one-to-many engine.

## Nicolas Cole's memorable writing framework (when an idea feels formless)

Every piece is built as Framework + Story + Actionable Advice, in that
order. The framework names the takeaway. The story anchors it in memory.
The advice closes with something to do.

This is a script structure, not a brainstorm engine. Reach for it when
the user has an idea but can't shape it into a 30 to 60 second piece.

Paired structural rule (the 1/3/1): one declarative opening sentence
(the hook), three middle sentences that build the argument, one punchy
closing sentence. Useful as a writing prompt for the script's spine.

## Dakota Robertson's GAP (when the feed feels lopsided)

Three content types every account needs to balance across a week:

- **Growth.** Trending topics, famous-name leverage, big-reach formats.
- **Authority.** Lessons, case studies, tutorials, testimonials.
- **Personal.** Vulnerable stories, worldview, behind-the-scenes.

GAP is a checklist, not an allocation. Every week should hit all three.
A week with only Authority is forgettable. A week with only Personal
doesn't grow. A week with only Growth never builds trust.

Use it when: the user has plenty of ideas but they're all the same
flavor, or the account has stalled and you can diagnose which bucket
is missing.

## Dan Koe's APAG (when one piece needs to land hard)

A four-beat structure for any single piece:

- **Attention.** Agitate the problem.
- **Perspective.** State the wrong view most people hold.
- **Advantage.** Explain why the correct view wins.
- **Gamify.** End with steps and a small challenge.

APAG pairs with a one-piece-per-week model: one validated source idea
(drawn from a tweet that worked, a Medium headline, or a YouTube
outlier), one core asset built with APAG, then decomposed into derived
shorts across platforms.

## Jay Clouse's PARTS (when the user is building a brand, not just videos)

A higher-level model for a creator business, not a per-video framework.
Useful when the user is asking about positioning, not ideation.

- **Premise.** Specific, defensible purpose. Pair with a "spiky point of
  view" via the XY pitch: "This is a project about X. Unlike other
  projects about X, only we Y."
- **Attention.** Discovery channels and owned channels.
- **Revenue.** Direct and indirect models.
- **Trust.** Earned via consistent overdelivery in owned channels.
- **Systems.** Automate the repeatable.

Useful idea hook from Clouse: treat every published piece as an
experiment. Observe, experiment, iterate. Ideas are tested, not picked.
The system rewards volume of tested attempts over polish on any single
attempt.

## Ben Meer's five storytelling slots (when an idea needs a vehicle)

Five named story shapes, each an idea slot you can drop a topic into:

- **Brand story.** Donald Miller's StoryBrand beats.
- **Interview story.** STAR (Situation, Task, Action, Result).
- **Customer success story.** Before, intervention, after.
- **Founder story.** Hero's-journey beats.
- **Company story.** The Timeline Method (milestones in order).

Useful for businesses and personal brands where ideas tend to be
content-rich but format-poor.

## Marie Forleo's SERVE (when the user wants a year of prompts in one session)

Five prompt categories. Forleo has used these to generate 100+ prompts
for 365 days of ideas:

- **Solve a problem.** Mine comments, Quora, Reddit, forums.
- **Explain your industry.** Translate insider knowledge for outsiders.
- **Reveal your secrets.** Personal stories, behind-the-scenes.
- **Visualize.** Help the audience imagine their future.
- **Embrace your quirks.** Mine personal memories and quirks.

Use it when: the user wants a long-horizon idea bank, not a single batch.

## MrBeast's quantitative idea scoring (when the user wants to filter, not generate)

From the leaked production memo. Every idea is scored pre-production
against four metrics:

- **CTR.** Predicted click-through rate of the title and thumbnail.
- **AVD.** Predicted average view duration.
- **AVP.** Predicted average view percentage.
- **Wow factor.** Gut excitement, the only qualitative metric.

Two rules that come with the system:

- **Redundancy.** If an idea is 50/50 to land, write a backup. Every
  load-bearing idea has a backup, every load-bearing component of an
  idea is over-engineered.
- **Personal excitement gate.** A flat idea dies before production. If
  the creator isn't excited to film it, it doesn't get made.

Most useful as a filter sitting on top of any of the generation engines
above. See `idea-evaluation.md` for the full filtering pass.

## Paddy Galloway's three-source funnel (when ideas need provenance)

Ideas come from three places, every week:

- **Internal.** Your own past outliers.
- **External.** Competitor outliers in your niche or adjacent niches.
- **Packaging-driven.** A new title or thumbnail concept that suggests
  a video.

Galloway's "borrow formats across niches" rule: a format that worked in
coding can be adapted to design, golf, or space if the underlying
mechanic (comparison, transformation, deadline) transfers. Most "new"
ideas in any niche are imported formats.

Pairs cleanly with `mining.md`, which covers how to actually find the
internal and external outliers.

## Which one to reach for

- User wants volume now: **content matrix.**
- User has one strong topic and wants 5 angles: **Ship 30 angle set** or
  **repurposing**.
- User's feed is lopsided in flavor: **GAP.**
- User is building positioning, not videos: **PARTS.**
- User has plenty of raw material but no shape: **memorable writing** or
  **Ben Meer's slots.**
- User wants a year of prompts: **SERVE.**
- User wants to filter an existing list, not generate more:
  **MrBeast quantitative + Galloway funnel + `idea-evaluation.md`.**
